Mutation Testing

powered by latest testing framework PIT

About mutation testing

  • Basically, it tests the quality of your tests.

  • Faults (or mutations) are automatically seeded into your code, then your tests are run. If your tests fail then the mutation is killed, if your tests pass then the mutation lived.

How PIT Works

  • PIT runs your unit tests against automatically modified versions of your application code.

  • PIT applies a configurable set of mutation operators (or mutators) to the byte code generated by compiling your code.

  • PIT will run your tests using this mutant instead of the unmutated class.

Better procedure

Exclude Equivalent Mutations:

  1. The resulting mutant behaves in exactly the same way as the original
  2. The resulting mutant behaves differently but in a way that is outside the scope of testing.

    }

Alternative Mutators

PIT provides 4 alternative groups of mutators

Better Testing strategy

  • Start from finding strong pseudo-tested methods.

1. strong pseudo-tested methods:

No matter the return values of a method, the tests still passes.(easy)

2. weak pseudo-tested methods:

The tests passes with at least one modified value.(hard)

3. fully tested methods:

The tests fail for all mutations and thus can be considered rock-solid!(almost impossible)

Extreme Mutation Testing

  • In Extreme Mutation testing, the whole logic of a method under test is eliminated.

void mutation operator

class A {

  int field = 3;

  public void Method(int inc) {
    field += 3;
  }

}

the mutation operator will generate:

class A {

  int field = 3;

  public void Method(int inc) { }

}

null mutation operator

class A {
    public A clone() {
        return new A();
    }
}

this operator will generate:

class A {
    public A clone() {
        return null;
    }
}

empty mutation operator

class A {
  public int[] getRange(int count) {
    int[] result = new int[count];
    for(int i=0; i < count; i++) {
      result[i] = i;
    }
    return result;
  }
}

will become:

class A {
  public int[] getRange(int count) {
    return new int[0];
  }
}

Constant mutation operator

class A {
    int field;

    public int getAbsField() {
        if(field >= 0)
            return field;
        return -field;
    }
}

this operator will generate:

class A {
    int field;

    public int getAbsField() {
        return 3;
    }
}
